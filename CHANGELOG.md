# 変更履歴

このプロジェクトの主な変更はこのファイルに記録します。形式は Keep a Changelog に基づきます。

## [Unreleased]

### Changed

- ディスカバーの更新順・新着・人気を待ち時間なく切り替えられるよう、3タブの先頭ページを起動時に並行取得してタブ別に保持し、非表示タブの先頭表紙も低優先度で先読み。
- ディスカバーの追加ページを待ち時間なく表示できるよう、次ページのデータと先頭表紙をバックグラウンド取得し、準備完了後にだけ「さらに読み込む」を表示してクリック直後から次のページを連鎖先読み。

### Fixed

- API側のCDN追加・切り替えで画像が表示不能にならないよう、ランタイム検証を通過したHTTPS画像URLをホスト名に依存せず表示・取得できる構成へ変更。
- DNSが停止した旧 `imfaclub.com` 系CDNを参照する章も読めるよう、APIから受け取った画像URLを同じパスの現行 `ihlv1.xyz` 系ホストへ正規化。

## [0.6.0] - 2026-07-22

### Added

- 作品の長さを直感的に絞り込めるよう、API実測分布に基づくコモンからレジェンダリーまでの5段階チャプター数フィルターを追加。

### Changed

- 検索画面で作品探しに集中できるよう、技術メタデータに加えて利用頻度の低いシリーズステータス、リリース年、API関連順を削除し、ジャンル、チャプター数、実用的な並び順へ整理。

### Fixed

- Android版でTauriの初期アイコンが表示されないよう、正規のアプリアイコンをAPKビルド前にAndroidリソースへ同期する処理を追加。

## [0.5.4] - 2026-07-22

### Added

- 縦読みの章末ナビゲーションへ、現在位置をチャプター数と全体比率で示す表示を追加。

### Fixed

- ピンチ操作とダブルタップが競合し、二本指で触れた際に表示が上下へ揺れる問題を修正。
- ピンチ中の二本指を安定して追跡し、指の中心位置を維持したまま拡大・縮小するよう修正。
- ダブルタップ時にタップ位置ではなく左上付近が拡大される問題を修正。

## [0.5.3] - 2026-07-22

### Changed

- 先頭画像の下引き再読み込みが操作量と結果を視覚的に把握できるよう、引いた距離に応じて画像を下へ追従させ、回転インジケーターを再取得完了まで表示するFirefox風のアニメーションへ変更。

### Fixed

- 縦読みでの左右スワイプによるチャプター移動が、意図した進行方向と逆になっていた問題を修正。

## [0.5.2] - 2026-07-22

### Changed

- ビューア操作ヘッダーの自動表示・非表示で漫画画像の位置や表示倍率が変わらないよう、ヘッダーを画像表示領域の前面へ重ねるレイヤー構成に変更。

## [0.5.1] - 2026-07-22

### Fixed

- 拡大した漫画の細部をWindowsとAndroidの両方で確認しやすくするため、マウスまたは1本指のドラッグで表示位置を縦横へ移動できない問題を修正。
- 縦読みのジェスチャーを直感どおり使えるよう、左右スワイプによる前後の章への移動と、先頭画像を下へ引いたときの再読み込みが動作しない問題を修正。

## [0.5.0] - 2026-07-22

### Added

- 漫画の細部を自然に確認できるよう、タップ位置を維持する1〜4倍のピンチイン・ピンチアウトと、ダブルタップによる2.5倍拡大・等倍復帰をビューアへ追加。
- Windows版でもマウスやタッチパッドだけで拡大できるよう、ツールバーの縮小・倍率リセット・拡大ボタン、タッチパッドのピンチ操作、`Ctrl++`・`Ctrl+-`・`Ctrl+0`を追加。

### Changed

- 拡大中の意図しないページ送りを防ぐため、1本指操作を画像の縦横移動へ切り替え、ページ送りボタンと章末ナビゲーションを一時的に隠すよう変更。

## [0.4.0] - 2026-07-22

### Changed

- ページ送りと章移動の待ち時間を最小化するため、表示位置の周辺、現在章全体、次章全体、次々章の冒頭、前章末尾の順に画像をアグレッシブに先読みするよう変更。
- 読みたいページをバックグラウンド取得に待たせないよう、すべての画像取得を同時8件までの優先度付き共通キューへ統合し、表示画像を常に先読みより優先するよう変更。
- 未読作品の読み始めをさらに速めるため、詳細表示時に最古章と最新章の先頭8ページずつを永続画像キャッシュへ先読みするよう変更。

## [0.3.2] - 2026-07-22

### Fixed

- 右綴じのページ送りと一貫するよう、章末ナビゲーションも次章を左、前章を右へ配置。

## [0.3.1] - 2026-07-22

### Changed

- 読書履歴とお気に入りをまとめた画面の役割が伝わるよう、ナビゲーションの「ヒストリー」を「ライブラリ」へ変更。

## [0.3.0] - 2026-07-22

### Added

- Windows版ビューアで画面を広く使えるよう、ヘッダーのボタンとF11キーによる全画面表示を追加。

### Changed

- ヒストリー画面を「マイライブラリ」、アップデート画面を「更新センター」とし、履歴・お気に入り・章通知・アプリ更新を含む実際の用途に合わせて説明文も変更。
- 未読作品の詳細を開いた時点で、最古章と最新章の先頭2ページずつを画像キャッシュへ先読みし、どちらから読み始めても初期表示を待ちにくく改善。
- 検索画面で頻用するジャンル・連載状態・並び順を常時表示し、ジャンル候補・状態・人気順・更新順・最新章順をワンクリックで即時適用できるクイックボタンを追加。低頻度のメタデータ条件だけを詳細アコーディオンへ整理。
- CIの品質検査・Windows・Androidを同時開始し、ReleaseではAndroid署名ビルドをWindowsと並行して成果物経由で合流するよう変更。BunとGradleの依存・ビルドキャッシュも再利用し、検証範囲を保ったまま待ち時間を短縮。
- 縦読みで表示中ページの前後と冒頭10ページを優先的に先読みし、画像CDNへの同時取得数を8へ広げて、読み進める速度へ追従しやすく変更。
- ビューア以外でブラウザー履歴による意図しない自動スクロールが発生しないよう、履歴のスクロール復元と全体のスムーズスクロールを無効化。

### Fixed

- ビューアのページ送りで矢印キーやページボタンを操作するたびに、非表示中のヘッダーと操作ヒントが再表示されないよう修正。
- 現在確認できる `s2.ihlv1.xyz`、`s4.ihlv1.xyz` と今後追加される同一ドメイン配下の画像CDNを取得できるよう、HTTP権限とCSPを `ihlv1.xyz` のHTTPSサブドメイン全体へ限定して許可。
- 右綴じの章移動方向と一致するよう、ビューアヘッダーの次章ボタンを左、前章ボタンを右へ修正。

## [0.2.0] - 2026-07-22

### Added

- 登録した作品を履歴と同じ画面で確認・解除できるよう、ヒストリーの下へお気に入り一覧を追加。

### Changed

- 縦読みの画面外先読み範囲を大幅に広げ、冒頭6ページを即時取得しつつ、同時通信数を制限して読み進める速度へ追従しやすく変更。
- 検索・ヒストリー・アップデートから端末の戻る操作を行うと常にディスカバーへ戻るよう、タブ移動をWebView履歴へ反映。
- お気に入りの保存形式を作品IDだけの配列から作品IDとタイトルの組へ更新し、既存データを自動移行して不足タイトルを作品情報から補完するよう変更。
- アップデート画面では章通知の下へアプリ更新を配置し、更新案内を「ボタンを押して最新版を確認します」へ簡潔化。
- 同じ漫画ページを開き直した際の待ち時間と通信を減らすため、画像をURL単位でメモリLRUへ保持し、進行中の取得を共有しつつ、対応環境では容量制限付きの永続キャッシュも再利用するよう変更。

### Fixed

- 漫画画像の通信失敗とタイムアウトを自動再試行し、画像デコード失敗も一度取り直すよう修正。回復できない場合は、黒い枠の代わりに再読み込みボタンを表示。
- CDNが画像のContent-Typeを欠落・誤設定していても、PNG・JPEG・GIF・WebP・AVIFの実データを検査して安全に表示できるよう修正。
- ヒストリーから縦読みビューアを再開した際、保存済みの読了割合に対応するページまで自動スクロールするよう修正。

## [0.1.1] - 2026-07-22

### Changed

- 読書中の移動を滑らかにするため、履歴から保存済みの章へ直接入り、章一覧の新旧順を切り替え、ビューアのヘッダーと章末から前後の章へ移動できるよう変更。
- 画像待ちを減らすため、縦読みの画面外先読み距離を広げ、ページ読みでは近接ページをマウントしたまま先読みするよう変更。

### Fixed

- Windows本番版でアプリと同時にコンソールウィンドウが開かないよう修正。
- Androidの戻るジェスチャーと端末の戻るボタンが、詳細・ビューアの一つ前の画面へ戻るようSPAの画面遷移をWebView履歴へ反映。

## [0.1.0] - 2026-07-21

### Added

- 非公式APIから作品一覧、検索、作品詳細、章画像を安全に取得して閲覧できる、n-mgram の初期デスクトップ版を追加。
- 大量画像の同時読み込みを抑える縦読み、キーボード操作に対応したページ読み、表示サイズ切り替えを追加。
- 途中から読書を再開できる読書位置と、お気に入りを端末内へ保存する機能を追加。
- API仕様変更、タイムアウト、アクセス制限を区別し、復旧方法を判断できるエラー表示を追加。
- 読んだ作品名、章、章内の進捗率、最終閲覧日時を一覧でき、作品単位で削除できる読書履歴を追加。
- 読書履歴の作品について最新章を確認し、未読の新しい章をまとめて開ける通知センターを追加。
- 作品APIで利用できる全メタデータを対象に、文字列、数値範囲、日付範囲、連載・表示状態を組み合わせて絞り込み、任意項目で昇順・降順に並べ替えられる詳細検索を追加。
- ディスカバー、検索、ヒストリー、アップデートを独立ページ化し、履歴の一括削除を追加。
- 署名済みGitHub Releaseを確認し、アプリ内で更新・再起動できるTauri updaterを追加。
- GitHub Actionsによる品質検査、Windowsコンパイル、タグ起点の署名付きRelease生成を追加。
- Android向けTauriプロジェクト、署名付きAPKの直接配布、アプリ内の新バージョン確認とブラウザ経由の更新導線を追加。
- 小画面では4ページを下部ナビゲーションへ配置し、縦持ちは1ページ、横持ちは右綴じ見開きで表示するモバイルレイアウトを追加。
- タッチ操作向けに、左タップ・左スワイプで次へ、右タップ・右スワイプで前へ、中央タップで操作UIを切り替えるページ操作を追加。

### Changed

- API側の互換判定に備え、TauriのネイティブHTTP通信から解析元アプリと同じUser-Agent、SDKバージョン、アプリバージョンヘッダーを送るよう変更。
- 日本漫画を本として自然な方向で読めるよう、ページ読みを表紙右配置の見開き表示にし、若いページを右、次ページを左へ置く右綴じ操作へ変更。
- 閲覧へ集中できるよう、操作が止まるとビューアのヘッダーと操作ヒントを自動的に隠し、ポインター・キーボード操作で再表示する挙動へ変更。
- 漫画や章を開き直すたびに表示を選び直さなくて済むよう、ビューアの縦読み・ページ読みと表示サイズの選択を端末内へ保存するよう変更。
- アプリ内の文言を日本語・英語の両方で利用できるよう整備。
- 既存ユーザーの読書位置とお気に入りを保ったまま履歴情報を追加できるよう、ローカルライブラリをバージョン2へ自動移行するよう変更。
- 非公式APIの仕様変更時に修正箇所をすぐ把握できるよう、接続先、偽装ヘッダー、レスポンス検証、データ正規化、画像処理を単一のAPI実装ファイルへ集約。

### Fixed

- Windows runnerの改行変換でRelease前のPrettier検査が失敗しないよう、checkout前にLFを固定し、既存タグを移動せず手動復旧できるRelease入力を追加。
- デスクトップ専用updater権限をプラットフォーム別capabilityへ分離し、Android向けRustビルドが未対応権限で停止する問題を修正。
- Newest一覧に更新日時が未設定の作品が含まれても一覧全体が取得失敗にならないよう、欠損日時のレスポンス検証を修正。
- Popularの長期連載作品に空の画像URLが混ざっても作品全体が取得失敗にならないよう、欠損ページだけを除外する章データ検証へ修正。
- 作品説明に含まれるHTMLタグや文字参照がそのまま画面へ出ないよう、安全なプレーンテキストへの正規化を追加。
- 翻訳者募集の案内画像が漫画ページへ混入しないよう、既知URLに加えて画像内容のSHA-256ハッシュを表示前に検証する除外処理を追加。
- ジャンル検索をAPI必須形式のオブジェクト配列で送信し、`psychological`などの検索がHTTP 400になる問題を修正。
